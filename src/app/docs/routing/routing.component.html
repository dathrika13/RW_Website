Routing

Redwood.ai Angular is having routing setup based on Angular-Router. The pre configured routing would make your job easy when dealing with any routing related changes. The routing is setup in a way to support lazy loading as well.

You can easily add, change or remove any route by simply making changes described below:

  1. Enable routing in the project using --routing while creating project.

  2. Add the entry for component in main application routing src/app/app-routing.module.ts E.g.

    {{'{'}}
    path: 'docs',
    canActivate: [AuthGuard],
    loadChildren: () => import('./docs/docs.module').then(m => m.DocsModule) //to load lazily
    or
    component: DocsComponent
    {{'}'}}

    Each of these properties are explained below:

    • path - relative path to component
    • loadChildren - module to load on this route
    • component - component to load on this route
    • canActivate - set of Guards which decides whether route can be actiavted or not. It restrict the non authenticated user to access this module.
